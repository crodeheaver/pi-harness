# Technical Research

## Procedure

1. Define the exact questions and distinguish required facts from optional background.
2. Prefer primary sources: official documentation, specifications, release notes, and source repositories.
3. Check publication/version dates and ensure findings apply to the versions in use.
4. Cross-check consequential claims. Treat package popularity, search ranking, and generated summaries as weak evidence.
5. Preserve source URLs alongside claims.
6. If web tools are unavailable, state the limitation instead of inventing current facts.

## Output

Summarize the recommendation, supporting evidence, tradeoffs, and unresolved uncertainty. Include a compact source list with access dates when recency matters.
